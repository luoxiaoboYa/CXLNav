# CXSearch 书签分类与标签设计

日期：2026-04-24

## 目标

CXSearch 需要同时支持平台内的高效筛选，以及与浏览器书签的双向导入 / 导出。浏览器书签天然是多级文件夹结构，而平台内日常查找更适合一级分类和标签筛选。因此设计上将“放在哪里”和“怎么使用”拆开。

## 核心结论

采用三类信息：

- 分类：一级主分类，用于主导航和大范围筛选。
- 书签路径：分类下的有序层级路径，用于表达浏览器书签目录结构。
- 普通标签：扁平标签，用于表达用途、状态、频率和场景。

一句产品规则：普通标签用于“怎么用”，书签路径用于“放哪里”。

## 数据语义

示例站点：

```ts
{
  title: 'Vue Router',
  url: 'https://router.vuejs.org',
  category: '开发',
  bookmarkPath: ['前端', 'Vue', '官方'],
  tags: ['高频', '学习', '文档'],
  sourcePath: ['开发', '前端', 'Vue', '官方'],
  source: 'browser-import'
}
```

字段含义：

- `category`：一级分类，比如 `开发`。
- `bookmarkPath`：分类下的书签路径，比如 `前端 / Vue / 官方`。
- `tags`：普通标签，比如 `高频`、`学习`、`文档`。
- `sourcePath`：导入时保留的完整原始浏览器路径，用于详情展示、编辑回显和来源追溯。
- `source`：站点来源，比如浏览器导入、手动新建、插件收藏。

## 导入规则

浏览器书签路径：

```text
开发 / 前端 / Vue / 官方 / Vue Router
```

导入后转换为：

- 分类：`开发`
- 书签路径：`前端 / Vue / 官方`
- 普通标签：默认不自动生成
- 原始路径：`开发 / 前端 / Vue / 官方`

导入预览页需要展示转换结果，并允许用户调整分类、书签路径和普通标签。

## 导出规则

导出回浏览器书签时，生成路径：

```text
分类 / 书签路径 / 站点名称
```

示例：

```text
开发 / 前端 / Vue / 官方 / Vue Router
```

推荐提供三种导出模式：

- 按原浏览器结构导出：优先使用 `sourcePath`，适合导入后未大幅调整的站点。
- 按 CXSearch 当前结构导出：使用 `category + bookmarkPath`，适合用户已在平台重新整理后的站点。
- 扁平导出到分类文件夹：只使用 `category / 站点名称`，适合需要干净导出结构的场景。

默认导出模式确认为“按 CXSearch 当前结构导出”，因为用户在平台内整理分类和书签路径后，通常希望导出整理后的新结构。


## 数据备份与浏览器书签导出的区别

导入 / 导出入口需要区分两类能力：

- 浏览器书签互通：用于和 Chrome、Edge、Firefox 等浏览器书签文件互相转换，重点保留文件夹结构、站点名称和 URL。
- CXSearch JSON 备份：用于平台自身数据恢复，完整保留分类、书签路径、普通标签、描述、来源、原始路径、分享状态、推荐状态和用户配置。

因此导入入口建议拆成：

- 从浏览器书签导入：解析浏览器文件夹结构，进入导入预览后再确认入库。
- 从 CXSearch JSON 备份恢复：恢复平台完整数据，不走书签路径转换逻辑。

导出入口建议拆成：

- 导出为浏览器书签：生成浏览器可重新导入的书签 HTML 文件，普通标签不参与目录结构。
- 导出为 CXSearch JSON 备份：生成平台完整备份文件，用于迁移和恢复。

## 列表筛选

列表页默认筛选方式：

```text
分类 → 普通标签
```

用户可切换为：

```text
分类 → 书签路径
```

默认使用普通标签，是因为日常查找更常按用途、状态和频率筛选。书签路径筛选用于整理目录结构、校对导出结构、批量迁移站点。

## 列表展示

简洁模式展示：

- 标题
- 描述
- 分类
- 普通标签

详细模式展示：

- 标题
- 描述
- 分类
- 书签路径
- 普通标签
- 来源 / 更新时间等辅助信息

详情页展示：

- 分类
- 书签路径
- 普通标签
- 原始导入路径
- 来源信息

## 新建与编辑

新建 / 编辑站点表单包含：

- 分类
- 书签路径
- 普通标签
- 标题、URL、描述等基础信息

当用户在列表中通过筛选后新建站点，表单自动继承当前上下文：

- 当前为 `分类 → 普通标签` 筛选时，自动带入分类和普通标签。
- 当前为 `分类 → 书签路径` 筛选时，自动带入分类和书签路径。

## 管理入口

后台管理建议拆成：

- 分类管理：管理一级分类。
- 标签管理：管理普通标签。
- 书签路径管理：管理分类下的路径树。
- 导入 / 导出：处理浏览器书签转换、预览、导出策略。

普通标签管理能力：

- 新增、重命名、删除
- 合并标签
- 查看使用次数
- 批量替换站点标签

书签路径管理能力：

- 在分类下新增路径
- 重命名路径节点
- 移动路径节点
- 合并路径
- 删除空路径
- 查看路径下站点数量
- 批量迁移站点到其他路径

## 关键边界

- 普通标签不参与浏览器目录导出，避免污染书签结构。
- 书签路径是有顺序的目录段，不等同于普通标签。
- 同名路径段可以存在于不同分类下，比如 `开发 / 官方` 和 `设计 / 官方`。
- 平台主分类保持一级，不做无限分类树。
- 原始导入路径应长期保留，除非用户明确选择重置来源结构。

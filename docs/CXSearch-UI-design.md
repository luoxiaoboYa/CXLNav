# CXSearch UI 设计稿 / 视觉规范

> 阶段定位：本文件用于“原型确认后、静态页面开发前”的 UI 设计确认。确认后再进入 UI 静态页面实现、前后端联动与插件开发。

## 1. 设计目标

CXSearch 是个人导航系统 + 浏览器插件，UI 应服务三个核心目标：

1. **快速找站**：搜索、分类、标签、常用站点必须高可见，减少用户翻找书签的成本。
2. **轻量整理**：添加、编辑、待整理、导入冲突处理要清晰，但不能像后台系统一样压迫。
3. **个人与推荐区分**：个人站点是私有工作台，推荐发现是公开资源库，二者在视觉上要能一眼区分。

设计关键词：**星图、探索、沉浸、清晰、可整理**。

## 2. 视觉方向

### 2.1 整体风格

- 已确认方向：`K6 太空探索 / 网址星图`。
- 基调：深空背景 + 星云紫 + 星光青，偏“网址星图 / 探索式个人导航”。
- 形态：星球核心、轨道线、星标卡片、悬浮信息块；保持清晰层级，不做过度装饰。
- 信息密度：PC 端中等密度；移动端优先保留标题、搜索、核心操作。
- 交互反馈：状态变化用星光高亮、轨道描边、标签文字提示，不使用强弹窗打断普通流程。

### 2.2 K6 默认主题

- 页面标题：700
- 卡片标题 / 按钮 / 重要状态：700
- 正文：400
- 次级元信息：400 或 500

- 虚线提示面板：`1px dashed var(--color-border-strong)`。

- 普通卡片不使用重阴影，避免后台感过强。

## 5. 栅格与响应式

### 5.1 断点

| 断点 | 范围 | 设计策略 |
| --- | --- | --- |
| Desktop | `>= 1120px` | 顶部导航四栏；内容双栏或三栏 |
| Tablet | `860px - 1119px` | 导航换行；管理中心侧栏可上置 |
| Mobile | `< 860px` | 单列布局；操作按钮换行；卡片简化 |
| Small Mobile | `< 480px` | 隐藏弱信息；按钮满宽或双行 |

### 5.2 页面容器

- 全局内容内边距：PC `24px`，移动 `16px`。
- 主要内容最大宽度建议：`1280px`，居中展示。
- 管理中心：PC 左侧导航 `260px` + 右侧自适应；移动端改为顶部横向/换行按钮组。

## 6. 组件设计规范

### 6.1 顶部导航

用途：全站模块切换、搜索入口、主题切换、登录入口。

视觉：

- 高度不固定，PC 推荐 `64px - 72px`。
- 背景继承页面暖白，底部 `#d7d2c6` 分割线。
- 品牌 `CXSearch` 左侧固定，字重 700。
- 导航链接使用 pill 按钮样式。
- 当前模块激活态：浅绿底 + 主色文字。

交互：

- 移动端导航自动换行，不做汉堡菜单作为 MVP 默认方案。
- 搜索按钮后续可展开全局搜索浮层。

### 6.2 搜索框

用途：首页、我的站点、推荐发现。

视觉：

- 大搜索框：高度 `48px - 56px`，圆角 `18px`。
- 输入区白底，外层可使用浅色面板包裹。
- 搜索范围切换使用 segmented control：`全部 / 我的站点 / 推荐发现`。

状态：

- Focus：主色边框 + 浅绿投影或背景。
- Empty：提示文案应说明可搜标题、详情、标签。

### 6.3 按钮

| 类型 | 用途 | 样式 |
| --- | --- | --- |
| Primary | 保存、确认、主 CTA | 绿底白字，边框同主色 |
| Secondary | 次操作 | 暖白底 + 默认边框 |
| Ghost | 关闭、取消 | 白底或透明底 + 默认边框 |
| Danger-light | 删除、移除 | 暖橙浅底 + 棕色文字，避免强红 |
| Disabled | 不可用 | 透明度 0.6，保留形状 |

按钮尺寸：

- 常规：`padding: 10px 12px`，圆角 `12px`。
- 小按钮：`padding: 8px 10px`。
- 移动端关键按钮可满宽。

### 6.4 卡片

#### 个人站点卡片

- 白底卡片，默认边框，圆角 `18px`。
- 顶部显示标题 + 安全/整理状态。
- 中部显示描述、分类、标签。
- 底部显示打开、编辑、分享申请等操作。
- 简洁模式只保留：标题、图标、链接/打开按钮、常用状态。
- 详情模式展示：描述、分类、标签、书签路径、共享状态、最近打开。

#### 推荐站点卡片

- 与个人卡片区分：可使用蓝色/浅绿渐变角标或来源标签。
- 必须标注来源：`系统推荐 / 用户分享 / 热门内容`。
- 平台标签与个人标签视觉区分：平台标签用蓝灰色，个人标签用绿色。

#### 管理卡片

- 信息密度略高，但保持圆角与留白。
- 状态、数量、操作按钮分区明确。
- 列表项 PC 可三列：主体信息 / 状态 / 操作。

### 6.5 标签与状态 Chip

| 类型 | 样式 | 示例 |
| --- | --- | --- |
| 普通标签 | 浅绿底 + 主色文字 | 高频、学习 |
| 平台标签 | 浅蓝灰底 + 蓝灰文字 | 设计、AI |
| 待整理 | 暖橙浅底 + 棕色文字 | 待补标签、链接异常 |
| 成功状态 | 浅绿底 + 主色文字 | 已整理、已上架 |
| 中性状态 | 暖白底 + 灰绿文字 | 未分类、未提交 |

### 6.6 状态 Tab

用于：站点管理、待整理中心、分享推荐、导入预览。

视觉：

- 默认：暖白底 + 灰绿文字 + 默认边框。
- 激活：浅绿底 + 主色文字 + 主色边框 + 字重 700。
- 文案格式：`状态名 数量`，如 `待整理 3`。

交互：

- 点击即筛选，不跳页。
- 筛选为空时显示空状态，不隐藏 Tab。

### 6.7 表单

- Label 在上，输入框在下。
- 输入框高度约 `42px - 46px`。
- 必填字段在 label 后加 `*`，但不要使用强红，错误时再提示。
- 说明 / 详情使用 textarea，最少 4 行。
- 多标签选择 MVP 可用 multi-select，正式 UI 可演进为可搜索标签选择器。

错误提示：

- 表单顶部显示汇总错误。
- 字段级错误显示在输入框下方，暖橙色。
- 成功提示使用浅绿提示条。

### 6.8 弹窗

用途：新增/编辑站点、导入确认、删除确认。

视觉：

- 遮罩：`rgba(31, 37, 31, 0.42)`。
- 弹窗宽度：PC `min(760px, 100%)`。
- 圆角：`28px`。
- 内边距：`24px`。
- 最大高度：`calc(100vh - 48px)`，内部滚动。

交互：

- 右上角关闭，底部取消/确认。
- 移动端弹窗接近底部 Sheet，但仍保留完整表单。

## 7. 页面 UI 设计

### 7.1 登录 / 注册页

目标：完成邮箱密码注册登录，第三方授权仅作为二期占位。

布局：

- 顶部页面标题 + 简短说明。
- 状态提示面板：说明当前接口 / 账号状态。
- 双栏表单：左登录，右注册。
- 移动端改为单列，登录在前，注册在后。

视觉重点：

- 表单卡片使用 `24px` 圆角面板。
- 主按钮统一绿色。
- 二期 OAuth 按钮置灰，不抢主流程。

### 7.2 首页

目标：作为快速入口，展示搜索、常用访问、热门推荐、个人分类概览。

布局：

1. Hero 搜索区：大标题 + 搜索框 + 范围切换。
2. 常用站点：个人常用，高频访问，卡片较简洁。
3. 热门推荐：公开推荐内容，标注来源和热度。
4. 分类浏览入口：进入我的站点完整结构。

视觉重点：

- Hero 可使用浅绿渐变背景。
- 常用站点与推荐站点卡片样式区分。
- 简洁模式下首页卡片隐藏详情。

### 7.3 我的站点页

目标：个人站点浏览、筛选、打开、编辑。

布局：

- 顶部搜索 + 模式切换。
- 左侧或顶部分类导航。
- 内容区：状态概览 + 标签筛选 + 站点卡片列表。
- 移动端：分类导航变横向滚动或换行 Chip。

视觉重点：

- 个人站点卡片以白底为主。
- 归档、待整理、链接异常用状态 Chip 标出。
- 简洁/详情模式必须视觉差异明确。

### 7.4 站点详情页

目标：展示个人信息、公开推荐引用、共享状态。

布局：

- 标题区：站点名、链接、主操作。
- 详情信息：描述、备注、分类、标签、书签路径。
- 状态与整理：安全状态、整理状态、归档状态、共享状态。
- 推荐库引用：如果存在推荐内容，展示公开信息对照。

视觉重点：

- 私有信息和公开信息分区明确。
- “个人备注、个人标签不会公开”的提示需可见。

### 7.5 推荐发现页

目标：浏览系统推荐和用户分享内容，收藏到个人站点。

布局：

- 搜索 + 筛选：分类、平台标签、排序。
- 推荐卡片：来源、类型、热度、收藏数。
- 收藏确认面板：保存到分类、是否添加个人标签。

视觉重点：

- 推荐卡片与个人卡片区分，建议使用浅蓝/浅绿来源角标。
- 平台标签只作为建议，视觉上不要混同个人标签。

### 7.6 管理中心

目标：承载低频维护操作。

结构：

- 左侧子导航：站点管理、待整理中心、分类管理、标签管理、书签路径管理、分享/推荐管理、导入/导出、回收站等。
- 右侧内容面板：根据模块显示表格、卡片、状态 Tab。
- 移动端：左侧导航上置，按钮换行。

视觉重点：

- 管理中心整体仍保持轻量，避免传统后台沉重表格。
- 状态 Tab 是主要筛选形式。
- 危险操作使用二次确认，不直接强红。

#### 7.6.1 待整理中心

- 顶部规则 Chip：缺少说明、疑似重复、长期未打开、链接异常、登录合并冲突。
- 状态 Tab：全部、待整理、整理中、稍后处理、已完成。
- 任务卡片：优先级、来源、建议处理、当前状态、操作。
- “立即整理”进入整理中；“稍后处理”进入稍后处理；完成后进入已完成。

#### 7.6.2 分类管理

- 顶部维护说明 + 新增分类按钮。
- 分类卡片：名称、站点数、待整理数、默认导出路径。
- 操作：编辑、排序、删除。

#### 7.6.3 标签管理

- 顶部维护说明 + 新增标签按钮。
- 标签卡片：名称、用途、使用次数、合并建议。
- 操作：重命名、合并、删除。

#### 7.6.4 导入 / 导出

- 功能卡片：浏览器书签导入、浏览器书签导出、JSON 备份、JSON 恢复。
- 导入预览必须有状态 Tab：全部、可导入、疑似重复、链接异常。
- 冲突项显示处理策略，不直接写入。

### 7.7 插件弹窗页

目标：模拟浏览器插件快速收藏。

尺寸建议：

- 宽度：`360px - 420px`。
- 高度：内容自适应，避免超过 `620px`。

布局：

- 当前页面标题与 URL。
- 保存到分类。
- 书签路径。
- 标签。
- 详情说明。
- 保存按钮。

视觉重点：

- 更紧凑，按钮清晰。
- 插件不直接发起共享或推荐，只保存到个人站点。

## 8. 模式设计

### 8.1 简洁模式

用于快速访问。

显示：

- 站点标题
- 图标 / 首字母占位
- 链接域名
- 打开按钮
- 高频 / 常用状态

隐藏：

- 详情说明
- 备注
- 大量标签
- 书签路径

### 8.2 详情模式

用于整理和筛选。

显示：

- 标题、链接、图标
- 详情说明
- 分类、标签、书签路径
- 安全状态、整理状态、共享状态
- 最近打开、更新时间
- 编辑 / 分享 / 归档操作

## 9. 可访问性与可用性

- 所有按钮需要明确文本或 `aria-label`。
- 表单控件必须有 label。
- 状态不能只靠颜色表达，必须有文字。
- 文本对比度遵守 WCAG AA 基础要求。
- 键盘可操作：Tab 顺序从导航到内容再到操作区。
- 移动端点击区域不低于 `40px`。

## 10. UI 交付确认清单

确认 UI 后，进入静态页面开发前，需要逐项确认：

- [ ] 明亮主题色彩是否确认。
- [ ] 暗黑主题是否 MVP 实现，还是先保留 token。
- [ ] 顶部导航是否保持当前多按钮布局，还是移动端改汉堡菜单。
- [ ] 个人站点卡片与推荐站点卡片区分是否足够明显。
- [ ] 简洁模式和详情模式的信息取舍是否确认。
- [ ] 管理中心是否坚持卡片式管理，而不是表格后台风格。
- [ ] 插件弹窗尺寸和信息字段是否确认。
- [ ] 删除、合并、导入确认是否都需要二次确认弹窗。

## 11. 后续静态页面开发原则

UI 确认后再进入静态页面开发，开发时应遵循：

1. 先抽全局 CSS token，不在组件内重复硬编码颜色。
2. 先实现基础组件：按钮、输入框、状态 Chip、卡片、弹窗、Tab。
3. 再重构页面样式，确保 PC 与移动端一致。
4. 最后再恢复前后端联动，不在 UI 静态阶段继续扩业务功能。

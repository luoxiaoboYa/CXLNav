# CXSearch 后期需求池

日期：2026-04-28

这个文件用于统一记录后期升级、优化和新功能想法。新点子先进入这里，后续需要实现时再逐条细化为正式需求、设计文档和开发计划。

## 使用规则

- 每个点子先记录为一条 backlog。
- 暂不确定的想法也可以记录，状态标为“待细化”。
- 如果某条需求内容较多，可以单独拆成详情文档，并在这里保留链接。
- 进入开发前，需要补齐目标、范围、数据模型影响、接口影响、验收标准。

## 状态说明

- 待细化：只有方向或场景，还没有完整需求边界。
- 已记录：需求动机和初步范围已经整理过。
- 待设计：可以进入方案设计，但还没有开发计划。
- 待实现：方案和计划已明确，可以排期开发。
- 已完成：已经实现并验证。

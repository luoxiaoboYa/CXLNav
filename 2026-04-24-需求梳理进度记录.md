## 今天确认的重点

### 1. 书签分类与导入导出

已确认三类信息分工：

- 分类：一级主分类，用于主导航和大范围筛选。
- 书签路径：表达浏览器书签目录结构，用于导入导出。
- 普通标签：表达用途、状态、频率和场景，不参与浏览器目录导出。

导入 / 导出已拆成两类能力：

- 浏览器书签互通：处理浏览器书签 HTML 的导入和导出。
- CXSearch JSON 备份：用于平台自身完整数据备份和恢复。

浏览器书签导出默认模式已确认为：按 CXSearch 当前结构导出。

相关文档：

### 2. 共享与推荐边界

已确认最终模型：

```text
我的站点 → 共享申请 → 推荐发现
```

核心结论：

- 插件只做快速私人收藏。
- 插件不直接发起共享或推荐。
- 共享只能在主站从“我的站点”发起。
- 推荐发现只展示系统推荐和审核通过的用户共享内容。
- 热门推荐不是独立来源，只是推荐发现中的排序或标记。

相关文档：

- `docs/superpowers/specs/2026-04-24-sharing-recommendation-design.md`

### 3. URL 安全与爬取防护

已确认 URL 风险边界：保存 URL 字符串风险相对可控，服务端主动访问用户提交 URL 才是主要风险点。

已定安全策略：

- 私人收藏允许先保存，默认状态为未校验。
- 未校验、风险、无法访问的站点不能发起共享申请。
- 自动补全只做轻量抓取，不执行 JavaScript，不保存原始 HTML。
- URL 抓取应放在独立 worker / sandbox 中。
- 禁止访问 localhost、内网 IP、云元数据地址和非 http/https 协议。
- 共享申请必须经过自动安全校验和后台审核。
- 风险站点不能进入推荐发现。

相关文档：

- `docs/superpowers/specs/2026-04-24-url-safety-design.md`

### 5. 账号体系与权限

账号体系采用：未登录也能本地使用，登录后云同步。

未登录可用：

- 添加个人站点到本地。
- 本地搜索。
- 分类 / 书签路径 / 普通标签管理。
- 浏览推荐库。
- 从推荐库收藏到本地。
- 浏览器插件本地收藏和本地搜索。

登录后解锁：

- 云同步和跨设备使用。
- 共享站点到推荐库。
- 发布推荐理由。
- 评论 / 举报。
- 管理我的共享。
- 数据备份到云端。

本地与云端数据合并规则：

- 登录后自动合并本地站点和云端站点。
- 以规范化 URL 作为主要去重依据。
- 普通标签取并集。
- 最近使用时间取最新。
- 收藏时间取最早。
- 共享状态以云端为准。
- 标题、分类、备注、书签路径冲突不阻塞登录，进入待整理中心。

### 6. 浏览器插件定位

插件定位为轻量工具：快速收藏、快速搜索、快速打开。

插件第一版能力：

- 快速收藏当前页面。
- 右键菜单收藏链接。
- 插件弹窗快速搜索已收藏站点。

插件明确不做：

- 不做共享到推荐库。
- 不做审核。
- 不做复杂管理。
- 不填写推荐理由。

共享推荐必须回到 Web 站点完成。插件收藏成功后只提示：

```text
已保存到我的站点
[打开 CXSearch] [继续浏览]
```

未登录时插件保存到本地；登录后同步到云端。离线或同步失败时，先保存到本地待同步队列。

### 7. 个人站点与公开推荐内容分离

已确认个人站点和公开推荐内容是两套数据：

- 个人站点属于用户自己的收藏、分类、书签路径、普通标签和备注。
- 公开推荐内容属于推荐库，包含公开标题、链接、分类、标签、描述、推荐理由等。
- 个人备注、个人书签路径、个人标签、导入来源路径、使用频率等不公开。

删除个人站点时，不自动删除已共享的公开推荐内容。删除时应提示：

```text
这个站点已共享到推荐库。
删除个人站点不会删除公开推荐内容。

[仅删除我的站点] [前往共享管理] [取消]
```

如果用户想处理公开内容，应进入 `我的共享 / 分享管理`：

- 可撤回自己的推荐理由。
- 可编辑自己的推荐信息。
- 可申请下架自己提交的推荐对象。
- 如果推荐对象已有其他用户互动或贡献，不能由单个用户直接删除整体，需要进入复审或下架流程。

### 8. 共享触发时机

共享不是新增站点时的必填流程，而是保存个人站点后的可选动作。

推荐触发入口：

- 新增站点保存成功后提示 `共享到推荐库`。
- 站点详情页按钮 `共享到推荐库`。
- 我的站点列表更多操作里 `共享 / 管理共享`。
- 插件收藏后只提供 `打开 CXSearch`，不在插件内共享。

共享流程独立于个人站点编辑流程，发起共享时再选择：

- 分享类型：主站点 / 功能站点。
- 实名 / 匿名。
- 公开推荐标题、描述、分类、标签。
- 推荐理由或使用原因。

### 9. 推荐库对象：主站点与功能站点

推荐库包含两类公开对象：

- 主站点：表示一个网站整体，例如 `GitHub`、`Figma`、`Notion`。
- 功能站点：表示某个网站下的具体功能、页面、项目、模板或文档，例如 `GitHub Trending`、`Figma Community`、`openai/codex 仓库`。

已确认规则：

- 主站点和功能站点内容结构相近，都是独立推荐内容。
- 功能站点可以收录到主站点下，但这种收录只发生在公开推荐库中。
- 用户个人站点里不显示这些公开聚合关系，个人数据仍按个人分类、书签路径、普通标签组织。
- 功能站点可以独立存在。
- 如果用户分享功能站点时主站点不存在，系统默认创建主站点草稿，并将功能站点关联到该草稿。
- 功能站点拥有独立分类和标签；若已有主站点，创建时默认继承主站点分类和通用标签，但用户可修改。
- 功能站点保存后以自身分类和标签为准，主站点后续修改不自动覆盖功能站点。

搜索结果也按这种思维展示：

```text
主站点
- GitHub

功能站点
- GitHub Trending
- openai/codex 仓库
```

### 10. 共享去重、免审与推荐理由

共享时先检查规范化 URL。

完整功能链接已存在：

- 不允许重复创建同一个公开功能站点。
- 引导用户查看已有详情页。
- 用户可以补充推荐理由 / 评论。

同域名不同路径：

- 允许作为新的功能站点分享。
- 如果主站点存在，收录到主站点下。
- 如果主站点不存在，系统自动创建主站点草稿。

审核规则：

- 新主站点共享需要审核。
- 主站点不存在时创建的主站点草稿需要后续完善 / 审核。
- 如果主站点已存在且已通过审核，用户分享该主站下的功能站点时，默认免人工审核，提交后直接发布。
- 免审不等于无校验，仍需 URL 规范化、唯一性校验、必填信息校验和基础风控。
- 功能站点必须填写功能描述和推荐理由。

补充推荐理由规则：

- 用户可选择实名或匿名。
- 默认通过自动风控后直接展示。
- 命中敏感词、广告、重复、过短、链接刷屏等规则时进入待复审。
- 被举报后可自动隐藏并进入后台复审。
- 同一用户对同一推荐对象建议只保留一条主推荐理由，可编辑自己的内容。

### 11. 搜索核心模块

搜索模块定位不是普通链接搜索，而是：站点找回与整理中心。

目标是解决：

- 浏览器书签太多，找东西麻烦。
- 用户忘记站点当初收藏来做什么。
- 老收藏变成僵尸数据，但又不敢删除。
- 收藏缺少说明、标签、用途和使用记录。

搜索应回答两个问题：

```text
我找的是哪个？
它现在对我还有什么用？
```

搜索范围：

- 我的站点。
- 推荐站点。
- 全部。

搜索字段：

- 标题。
- URL / 域名。
- 分类。
- 书签路径。
- 普通标签。
- 一句话用途。
- 详细描述。
- 用户备注。
- 收藏来源。
- 推荐库说明。
- 用户推荐理由。

搜索结果卡片需要展示：

- 站点名称。
- 一句话用途。
- 命中原因。
- 分类 / 标签 / 书签路径。
- 最近使用 / 收藏时间。
- 整理状态。
- 快捷操作：打开、详情、补充备注、标记待整理、查看推荐库说明、归档、删除等。

搜索模块旁边应有固定入口 `待整理`，用于处理：

- 缺少说明。
- 长期未打开。
- 疑似重复。
- 链接疑似失效。
- 只有导入路径没有标签。
- 推荐库已有更完整说明。

### 12. 轻录入与渐进完善

已确认不能让用户填写过多内容。产品原则：

```text
用户只需要保存链接，系统尽量自动补全；用户只在需要时补充关键说明。
```

个人保存时：

- 只强制 URL。
- 用户备注可选。
- 标题、描述、favicon、分类建议、标签建议、书签路径建议、用途说明草稿由系统自动补全或建议。

信息来源：

- 网页自动信息：标题、描述、favicon、Open Graph 信息。
- 用户上下文：当前分类、筛选标签、当前书签路径。
- 导入信息：浏览器书签路径、浏览器书签标题、导入时间。
- 推荐库信息：已有主站点说明、功能站点说明、其他用户推荐理由、平台标签。
- 用户主动补充：我的备注、普通标签、用途说明、推荐理由。

信息不完整不阻塞保存，而是进入待整理状态：

- 说明待补充。
- 标签待整理。
- 路径待确认。
- 推荐库可补充。

只有用户主动共享到推荐库时，才强制填写推荐描述和推荐理由。

### 13. 导入 / 导出修正规则

导入方式修正为：

1. 标准导入：保留书签路径，推荐默认。

```text
第一层 → 分类
后续层级 → 书签路径
完整路径 → 原始路径
```

2. 简化导入：只保留一级分类。

```text
第一层 → 分类
后续层级不进入书签路径
完整路径仍保留为原始路径
问题项进入待整理
```

不再使用“全部压缩为一级分类”这个说法，避免造成分类爆炸和重复混乱。

个人导入和公开推荐去重规则必须分开：

个人站点库：

- 同 URL 可以存在多条个人收藏。
- 默认保留浏览器书签原结构。
- 重复 URL 只提示，不强制合并。
- 用户可在导入预览或待整理中心选择合并、跳过或删除重复项。

公开推荐库：

- 主站点 / 功能站点采用唯一性规则。
- 完整功能 URL 已存在时不能重复创建。
- 用户只能补充推荐理由。

导入预览页发现重复时，应提示：

```text
发现 12 个重复链接。
默认保留原书签位置，导入后可在待整理中心合并。
```

重复链接处理默认选项：

```text
[保留多个收藏位置] 推荐
[自动合并为一个站点]
[跳过重复项]
```

# CXSearch 需求梳理进度记录

## 今天确认的重点

### 1. 书签分类与导入导出

已确认三类信息分工：

- 分类：一级主分类，用于主导航和大范围筛选。
- 书签路径：表达浏览器书签目录结构，用于导入导出。
- 普通标签：表达用途、状态、频率和场景，不参与浏览器目录导出。

导入 / 导出已拆成两类能力：

- 浏览器书签互通：处理浏览器书签 HTML 的导入和导出。
- CXSearch JSON 备份：用于平台自身完整数据备份和恢复。

浏览器书签导出默认模式已确认为：按 CXSearch 当前结构导出。

### 2. 共享与推荐边界

已确认最终模型：

```text
我的站点 → 共享申请 → 推荐发现
```

核心结论：

- 插件只做快速私人收藏。
- 插件不直接发起共享或推荐。
- 共享只能在主站从“我的站点”发起。
- 推荐发现只展示系统推荐和审核通过的用户共享内容。
- 热门推荐不是独立来源，只是推荐发现中的排序或标记。

相关文档：

- `docs/superpowers/specs/2026-04-24-sharing-recommendation-design.md`

### 3. URL 安全与爬取防护

已确认 URL 风险边界：保存 URL 字符串风险相对可控，服务端主动访问用户提交 URL 才是主要风险点。

已定安全策略：

- 私人收藏允许先保存，默认状态为未校验。
- 未校验、风险、无法访问的站点不能发起共享申请。
- 自动补全只做轻量抓取，不执行 JavaScript，不保存原始 HTML。
- URL 抓取应放在独立 worker / sandbox 中。
- 禁止访问 localhost、内网 IP、云元数据地址和非 http/https 协议。
- 共享申请必须经过自动安全校验和后台审核。
- 风险站点不能进入推荐发现。

相关文档：

- `docs/superpowers/specs/2026-04-24-url-safety-design.md`

## 下次建议继续梳理

建议继续按以下顺序：

1. 搜索模块：搜索范围、排序、历史、空状态、个人/推荐结果区分。
2. 登录与权限：未登录、已登录、管理员/后台角色能做什么。
3. 站点状态模型：私人收藏、共享申请、推荐发现、下架、取消共享之间的状态流转。
4. 后端接口与数据表：围绕站点、分类、标签、共享申请、推荐发现、安全校验结果建模。
5. 插件详细流程：右键入口、弹窗字段、同步失败、离线/未登录状态。

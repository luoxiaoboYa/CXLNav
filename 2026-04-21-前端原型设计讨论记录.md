# CXSearch 前端原型设计讨论记录

日期：2026-04-21

## 当前阶段

- 需求基准文档：`cxsearch最终版本.md`
- 当前聚焦子项目：前端 UI 原型
- 当前目标层级：高保真原型
- 当前状态：已完成第一轮范围收敛，未进入正式设计稿输出

## 已确认决策

- 本轮只做 `前端 UI 原型`，不展开后端和浏览器插件实现细节
- 原型保真度选择为 `高保真原型`
- 视觉方向选择为 `内容导航型`
- 首页主优先级选择为 `个人站点优先`
- 个人站点首页布局倾向为 `卡片浏览型`
- 首页管理强度选择为 `轻管理`
  - 首页以浏览为主
  - 新增、编辑、分类管理进入独立页面

## 已提出的结构方案

提出了 3 个首页结构方向：

1. 纯个人导航首页
2. 个人优先 + 发现穿插
3. 双首页并列型

当前推荐方案：

- `2. 个人优先 + 发现穿插`

推荐原因：

- 符合“个人内容优先”的核心判断
- 保留“内容导航型”的探索感
- 适合卡片浏览为主的首页体验
- 与“轻管理”策略不冲突

## 待确认事项

- 用户尚未明确确认是否采用方案 `2. 个人优先 + 发现穿插`
